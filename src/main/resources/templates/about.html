What are Use Case Points?

Use Case Points (UCP) is an estimation method primarily used in software development to predict the size and effort required for a software project based on use cases.

Developed by Gustav Karner in 1993 while working at Objectory AB (later acquired by Rational Software), it's particularly suitable for object-oriented development projects. The method provides a systematic approach to estimating project effort by analyzing actors, use cases, and various adjustment factors that impact development complexity.

Note: This application uses a simplified UCP formula: UCP = (UUCW + UAW) × VAF, where VAF (Value Adjustment Factor) replaces the traditional TCF and ECF components. This simplified approach maintains estimation accuracy while making the process more accessible and straightforward.

UCP Calculation Process

1 Identify and Classify Actors
Categorize actors as Simple (1), Average (2), or Complex (3)
2 Identify and Classify Use Cases
Categorize use cases as Simple (5), Average (10), or Complex (15)
3 Calculate Actor and Use Case Weights
Determine Unadjusted Actor Weight (UAW) and Unadjusted Use Case Weight (UUCW)
4 Determine Technical Factors
Assess 14 technical factors to calculate Total Degree of Influence (TDI)
5 Calculate UCP
UCP = (UUCW + UAW) × Value Adjustment Factor (VAF)
6 Estimate Effort
Effort (hours) = UCP × 20 hours per UCP

Key Formulas in UCP

TDI = Sum of all Technical Factors (rated 0-5 × weight)
VAF = 0.65 + (0.01 × TDI)
UCP = (UUCW + UAW) × VAF
Effort = UCP × 20 hours
Actor Classification
Simple (Weight: 1)

External system with a defined API

Example: External web service, database
Average (Weight: 2)

External system interacting through a protocol

Example: Person through a command-line interface
Complex (Weight: 3)

Person interacting through a graphical user interface

Example: User through a web interface
Use Case Classification
Simple (Weight: 5)

≤ 3 transactions, ≤ 5 analysis classes

Example: Login, simple CRUD operation
Average (Weight: 10)

4-7 transactions, 5-10 analysis classes

Example: Report generation, multi-step process
Complex (Weight: 15)

≥ 8 transactions, ≥ 10 analysis classes

Example: Complex processing, multi-system integration

Technical Factors for VAF Calculation

Factor Description Weight
T1 Distributed System 2.0
T2 Response Time/Performance Objectives 1.0
T3 End-User Efficiency 1.0
T4 Complex Internal Processing 1.0
T5 Reusability 1.0
T6 Easy to Install 0.5
T7 Easy to Use 0.5
T8 Portability 2.0
T9 Easy to Change 1.0
T10 Concurrency 1.0
T11 Security Features 1.0
T12 Third Party Access 1.0
T13 User Training 1.0
T14 Facilitate Change 1.0
Value Adjustment Factor (VAF) Formula: VAF = 0.65 + (0.01 × TDI)
Where TDI is the Total Degree of Influence, calculated as the sum of all technical factors (each rated from 0-5) multiplied by their respective weights.

Effort Estimation

Simplified Effort Estimation: In the current implementation, we use a fixed productivity factor of 20 hours per UCP for all projects. This simplification makes the estimation process more straightforward while still providing a reasonable basis for project planning.
Benefits of the Simplified UCP Method
  • More straightforward calculation with fewer variables
  • Focus on core technical factors that affect development complexity
  • Consistent productivity factor for more predictable estimates
  • Easier to apply across different project types
  • Reduced subjectivity in the estimation process
Formula Summary:
  • UCP = (UUCW + UAW) × VAF
  • VAF = 0.65 + (0.01 × TDI)
  • Effort = UCP × 20 hours
