What are Use Case Points?

Use Case Points (UCP) is an estimation method primarily used in software development to predict the size and effort required for a software project based on use cases.

Developed by Gustav Karner in 1993 while working at Objectory AB (later acquired by Rational Software), it's particularly suitable for object-oriented development projects. The method provides a systematic approach to estimating project effort by analyzing actors, use cases, and various adjustment factors that impact development complexity.

UCP Calculation Process

1 Identify and Classify Actors
Categorize actors as Simple (1), Average (2), or Complex (3)
2 Identify and Classify Use Cases
Categorize use cases as Simple (5), Average (10), or Complex (15)
3 Calculate Unadjusted Use Case Points (UUCP)
UUCP = Unadjusted Actor Weight (UAW) + Unadjusted Use Case Weight (UUCW)
4 Determine Technical and Environmental Factors
Assess 13 technical factors and 8 environmental factors
5 Calculate UCP
UCP = UUCP × Technical Complexity Factor (TCF) × Environmental Complexity Factor (ECF)
6 Estimate Effort
Effort (hours) = UCP × Productivity Factor (20-28 hours per UCP)

Key Formulas in UCP

UUCP = UAW + UUCW
TCF = 0.6 + (0.01 × Technical Factor Total)
ECF = 1.4 + (-0.03 × Environmental Factor Total)
UCP = UUCP × TCF × ECF
Effort = UCP × Productivity Factor
Actor Classification
Simple (Weight: 1)

External system with a defined API

Example: External web service, database
Average (Weight: 2)

External system interacting through a protocol

Example: Person through a command-line interface
Complex (Weight: 3)

Person interacting through a graphical user interface

Example: User through a web interface
Use Case Classification
Simple (Weight: 5)

≤ 3 transactions, ≤ 5 analysis classes

Example: Login, simple CRUD operation
Average (Weight: 10)

4-7 transactions, 5-10 analysis classes

Example: Report generation, multi-step process
Complex (Weight: 15)

≥ 8 transactions, ≥ 10 analysis classes

Example: Complex processing, multi-system integration
Benefits of UCP
  • Early estimation in development cycle
  • Based on use cases (common project artifacts)
  • Considers technical and environmental factors
  • Relatively simple to apply
  • Adaptable to different project types

Technical Factors

Technical factors capture the influence of various technical aspects on the project. Each factor is assigned a weight and rated from 0 (no influence) to 5 (strong influence).

ID Factor Weight Description
T1 Distributed System 2.0 The system involves distributed components or architecture
T2 Response Time/Performance 1.0 Performance requirements influence development
T3 End-User Efficiency 1.0 Optimization for user efficiency is important
T4 Complex Internal Processing 1.0 Complex algorithms or business logic
T5 Reusability 1.0 Code must be designed for reuse
T6 Easy to Install 0.5 Installation simplicity is important
T7 Easy to Use 0.5 Usability is a key requirement
T8 Portability 2.0 System must work on multiple platforms
T9 Easy to Change 1.0 System designed for easy maintenance
T10 Concurrency 1.0 System must handle concurrent users/processes
T11 Special Security Features 1.0 Security is an important consideration
T12 Direct Access for Third Parties 1.0 System provides access to external systems
T13 Special User Training Facilities 1.0 System requires special training features

Technical Complexity Factor (TCF) = 0.6 + (0.01 × TF)
Where TF is the sum of the weighted scores (weight × rating) for all technical factors.

Environmental Factors

Environmental factors capture the influence of team and project environment aspects. Each factor is assigned a weight and rated from 0 (very low) to 5 (very high).

ID Factor Weight Description Note
E1 Familiarity with Project 1.5 Team's familiarity with the project type Higher is better
E2 Application Experience 0.5 Team's experience with similar applications Higher is better
E3 Object-Oriented Experience 1.0 Team's experience with OOP Higher is better
E4 Lead Analyst Capability 0.5 Capability of the lead analyst Higher is better
E5 Motivation 1.0 Team's motivation level Higher is better
E6 Stable Requirements 2.0 Stability of requirements Higher is better
E7 Part-Time Staff -1.0 Use of part-time staff Higher is worse
E8 Difficult Programming Language -1.0 Complexity of programming language Higher is worse

Environmental Complexity Factor (ECF) = 1.4 + (-0.03 × EF)
Where EF is the sum of the weighted scores (weight × rating) for all environmental factors.

Productivity Factor Determination

The productivity factor (hours per UCP) is determined by counting significant environmental factors:

  • Count factors E1-E6 with score < 3
  • Count factors E7-E8 with score > 3

Based on the count of significant factors:

  • 0-2 significant factors: 20 hours/UCP (favorable conditions)
  • 3-4 significant factors: 28 hours/UCP (average conditions)
  • 5+ significant factors: 36 hours/UCP (difficult conditions)

UCP Calculation Process

Step 1: Identify and Weigh Actors
  • Classify each actor as Simple, Average, or Complex
  • Multiply by respective weights (1, 2, or 3)
  • Sum up to calculate UAW (Unadjusted Actor Weight)
Step 2: Identify and Weigh Use Cases
  • Classify each use case as Simple, Average, or Complex based on transaction count
  • Multiply by respective weights (5, 10, or 15)
  • Sum up to calculate UUCW (Unadjusted Use Case Weight)
Step 3: Calculate UUCP
  • UUCP = UAW + UUCW
Step 4: Evaluate Technical Factors
  • Rate each technical factor from 0-5
  • Multiply by respective weights
  • Sum to get TF (Technical Factor total)
  • Calculate TCF = 0.6 + (0.01 × TF)
Step 5: Evaluate Environmental Factors
  • Rate each environmental factor from 0-5
  • Multiply by respective weights
  • Sum to get EF (Environmental Factor total)
  • Calculate ECF = 1.4 + (-0.03 × EF)
Step 6: Calculate Final UCP and Effort
  • UCP = UUCP × TCF × ECF
  • Determine appropriate productivity factor
  • Effort = UCP × Productivity Factor

Benefits and Limitations

Benefits of UCP
  • Based on use cases, which are often created early in development
  • Takes into account both technical and environmental factors
  • Relatively simple to calculate compared to other estimation methods
  • Provides a structured approach to estimation
  • Well-suited for object-oriented projects
  • Can be calculated early in the project lifecycle
Limitations of UCP
  • Depends on the quality of use case documentation
  • Requires some subjective judgments in rating factors
  • May not be as accurate for non-OO projects
  • Needs calibration within an organization for best results
  • Doesn't account for all project activities (e.g., deployment, training)
  • Productivity factor may vary significantly between organizations
